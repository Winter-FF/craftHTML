Design Decisions

Module-based architecture (not flat routes)
Each feature (auth, tasks, workspaces) is self-contained with its own controller, service, and routes. This allows teams to work independently and makes testing easier.
Tradeoff: more boilerplate per module, slightly harder to see the full route table at a glance.
Prisma ORM (not raw SQL or TypeORM)
Type-safe queries with auto-generated client. Migration system is declarative. Developer experience prioritized over query optimization flexibility.
Tradeoff: complex queries require raw SQL via $queryRaw. Slight runtime overhead vs raw pg driver.
JWT + Redis for auth (not session-only)
JWTs allow stateless verification at the API gateway level. Redis stores refresh tokens and enables instant revocation (logout, password change).
Tradeoff: more infrastructure (Redis required). Token size adds ~500 bytes per request header.
